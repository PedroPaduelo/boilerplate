9 endpoints RESTful para busca e admin

Endpoints da API

GET /api/search
Public

Full-text search com filtros opcionais.

Query params: q (string), page, limit, type, status, tags
Response: { hits: [], total: number, page, limit }
GET /api/search/geo
Public

Busca por proximidade geográfica.

Query params: lat, lon, radius (km), limit
Response: { hits: [{_id, _score, _source, _geo}] }
GET /api/search/autocomplete
Public

Autocomplete para typeahead.

Query params: q (prefix), field, size
Response: { suggestions: [{text, score}] }
POST /api/search/index
Admin Only

Indexa ou atualiza um documento.

Body: { id, title, content, type, author, location?, tags[] }
Response: { indexed: true, id: string }
POST /api/search/bulk-index
Admin Only

Indexação em lote (multi documentos).

Body: Array<{ id, title, content, type, ... }>
DELETE /api/search/document/:id
Admin Only

Remove um documento do índice.

GET /api/search/analytics
Public

Estatísticas de busca (não implementado no schema?).

Schema do Documento Indexado

export const searchDocumentSchema = z.object({
  id: z.string().required(),           // UUID do documento
  type: z.enum(['post', 'page', 'user']).required(),
  title: z.string().required(),
  content: z.string().optional(),
  author: z.object({
    id: z.string(),
    name: z.string(),
    email: z.string()
  }).optional(),
  category: z.object({
    id: z.string(),
    name: z.string(),
    slug: z.string()
  }).optional(),
  tags: z.array(z.string()).default([]),
  status: z.enum(['draft', 'published', 'archived']).default('draft'),
  location: z.object({
    lat: z.number(),
    lon: z.number()
  }).optional(),                      // Para geo-search
  createdAt: z.date().default(Date.now),
  updatedAt: z.date().default(Date.now)
});

Riscos

Query Injection

DSL do OpenSearch pode ser injetado se não validado.

Mitigação: Validar schema Zod, never raw user DSL.

Caching

Sem cache layer entre API e OpenSearch.

Melhoria: Redis cache para hot queries.

Exemplos cURL

Busca full-text

curl -X GET "http://localhost:3333/api/search?q=react&limit=10"

Geo-search

curl -X GET "http://localhost:3333/api/search/geo?lat=-23.5&lon=-46.6&radius=50"

Indexar documento

curl -X POST http://localhost:3333/api/search/index \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "post-123",
    "type": "post",
    "title": "Introdução ao React",
    "content": "React é uma biblioteca...",
    "author": { "id": "u1", "name": "João", "email": "joao@email.com" },
    "tags": ["react", "frontend"],
    "status": "published"
  }'
