📋 Matriz RBAC (Role-Based Access Control)

👑 ADMIN

  • ✓ Criar/editar/deletar qualquer usuário
  • ✓ Indexar documentos de qualquer usuário
  • ✓ Ver todas as métricas e logs
  • ✓ Gerenciar roles e permissões
role: 'ADMIN'

👤 USER

  • ✓ Editar próprio perfil
  • ✓ Buscar documentos (read-only)
  • ✓ Indexar próprios documentos
  • ✓ Ver próprias métricas
  • ✗ Deletar outros usuários
role: 'USER'

🌍 PUBLIC

  • ✓ Registrar nova conta
  • ✓ Login com credenciais
  • ✓ Buscar (read-only, sem auth)
  • ✗ Indexar documentos
  • ✗ Editar qualquer recurso
não autenticado

⚠️ Riscos de Segurança Identificados

🔥

Deleção em Cascata

Deleção de um usuário remove todos os seus posts, arquivos e dados relacionados.

Impacto: Perda irreversível de dados
Mitigação: Implementar soft delete, backup automático, confirmação antes da deleção
⚡

Rate Limit Limitado

Rate limit apenas global por IP, sem Rate Limit por usuário autenticado.

Impacto: Ataque DoS via múltiplas contas
Mitigação: Adicionar rate limit por user ID após auth
🔓

Sem 2FA (Two-Factor Authentication)

Sistema não possui autenticação de dois fatores, apenas senha.

Impacto: Vulnerável a credential stuffing, phishing
Mitigação: Implementar TOTP (Google Authenticator) ou SMS/Email 2FA
✉️

Sem Verificação de Email

Usuários registrados podem usar o sistema sem confirmar_email.

Impacto: Contas falsas, spam, impossible to recover
Mitigação: Bloquear login sem verificação, token de confirmação
📜

JWT sem Refresh Token Rotation

Refresh tokens podem ser reutilizados se roubados (sem verificação de jti).

Impacto: Sessão prolongada não autorizada
Mitigação: Implementar jti, invalidar após uso, curta validade access token
🔍

Busca sem Limite de Resultados

Limite default de 20, mas pode ser elevado para 100 via query param.

Impacto: DoS via open search广度扫描
Mitigação: Max 50 hard coded, monitorar uso
