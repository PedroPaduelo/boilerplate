Riscos de Dados

🚨 Deleção em Cascata

Quando um User é deletado, todos os seus Post são automaticamente deletados (onDelete: Cascade).

Impacto: Perda total de conteúdo关联ado ao usuário.

⚠️ Senha em Texto Plano

O campo password da tabela users armazena senhas sem hash.

Mitigação recommended: Usar bcrypt.hash com salt rounds (10+).

📊 Índices Insuficientes

Apenas índices simples em authorId e categoryId.

Melhoria: Índices compostos para queries frequentes (ex: published + createdAt).

✅ Boas Práticas

@updatedAt automático, timestamps com @map para snake_case, unique constraints.

Schema bem projetado para um boilerplate.

Exemplos de Queries (Prisma)

Listar posts com autor

const posts = await prisma.post.findMany({
  where: { published: true },
  include: {
    author: { select: { id: true, email: true, name: true } },
    category: true
  },
  orderBy: { createdAt: 'desc' },
  skip: 0,
  take: 20
});

Criar post

const post = await prisma.post.create({
  data: {
    title: 'Novo Post',
    content: 'Conteúdo...',
    authorId: userId,
    categoryId: categoryId
  }
});

Count de posts por autor

const count = await prisma.post.count({
  where: { authorId: userId }
});

Deletar usuário (trigger cascata)

// Isto deleta o usuário E TODOS os seus posts
await prisma.user.delete({
  where: { id: userId }
});
